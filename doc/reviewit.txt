1. INTRODUCTION                                        *reviewit-introduction*

reviewit.nvim is a Neovim plugin that enables GitHub PR code review directly
in the editor. It provides:

- Base branch diff preview in a side pane
- Virtual text indicators for existing PR comments
- Create, view, and reply to PR review comments
- Suggest code changes with GitHub's suggestion blocks
- Navigate between comments
- Browse changed files with Telescope or quickfix
- View PR overview with title, description, labels, and comments
- Clickable GitHub references (#123) and URLs in floating windows
- @user and #issue completion in comment windows (blink.cmp / nvim-cmp)
- Gitsigns integration for diff base switching

==============================================================================
2. REQUIREMENTS                                        *reviewit-requirements*

- Neovim >= 0.10
- GitHub CLI (`gh`) installed and authenticated
- Git repository with a PR on the current branch

Optional:
- telescope.nvim for file picker with diff preview
- gitsigns.nvim for automatic diff base switching
- blink.cmp or nvim-cmp for @user / #issue completion

4. COMMANDS                                              *reviewit-commands*

:ReviewStart                                                  *:ReviewStart*
    Start review session for the current branch's PR. Detects the PR
    automatically using `gh pr view`, fetches comments, sets up
    extmarks, and switches gitsigns base to the PR base branch.
    Does not open the diff preview (use |:ReviewDiff| for that).

:ReviewStop                                                    *:ReviewStop*
    Stop review mode and clean up all preview windows, extmarks,
    autocommands, and restore original settings.

:ReviewToggle                                                *:ReviewToggle*
    Toggle review session on or off.

:ReviewDiff                                                    *:ReviewDiff*
    Toggle the diff preview window. Opens a side-by-side diff view
    showing the base branch version of the current file, or closes
    it if already open. Requires an active review session.

:ReviewComment                                              *:ReviewComment*
    Create a review comment on the current line (normal mode) or
    selected lines (visual mode). Opens a floating window for
    composing the comment. Press `<CR>` to save as GitHub pending review
    (visible on PR page with "Pending" status), `q` to cancel.
    Pending comments can be submitted as a review using |:ReviewSubmit|.

:ReviewViewComment                                      *:ReviewViewComment*
    View existing review comments on the current line in a floating
    window. Press `r` to reply, `q` to close.

:ReviewFiles                                                *:ReviewFiles*
    List files changed in the PR. Uses Telescope with diff preview
    by default, falls back to quickfix if Telescope is not available.
    Configurable via `file_list_mode`.

:ReviewOverview                                            *:ReviewOverview*
    Show PR overview in a centered floating window. Displays title,
    state, author, labels, description, and issue-level comments.
    Requires an active review session.
    Press `c` to write a new comment, `R` to refresh, `q` to close.

:ReviewSuggest                                            *:ReviewSuggest*
    Suggest a code change on the current line (normal mode) or
    selected lines (visual mode). Opens a floating window pre-filled
    with a GitHub suggestion block containing the selected code.
    Edit the code inside the block, then press `<CR>` to save as
    GitHub pending review. Pending comments can be submitted as a
    review using |:ReviewSubmit|. The suggestion appears on the PR
    with an "Apply suggestion" button.

:ReviewListComments                                    *:ReviewListComments*
    List all PR review comments in a Telescope picker. Shows each
    comment location with author and body preview. Select an entry
    to jump to the file and line. Preview pane shows the full comment
    thread formatted as markdown. Requires an active review session.

:ReviewListDrafts                                        *:ReviewListDrafts*
    List all local draft comments in a Telescope picker. Shows each
    draft location with a body preview. Select an entry to jump
    to the file and line. Preview pane shows the full draft content.
    Press `d` to delete a draft. Note: Most comments are saved as
    GitHub pending review (see |:ReviewComment|), not local drafts.
    Requires an active review session.

:ReviewBrowse                                                *:ReviewBrowse*
    Open the PR in the browser using the cached PR URL.
    Requires an active review session.

:ReviewSubmit                                              *:ReviewSubmit*
    Submit all pending comments as a single GitHub review. This allows
    you to batch comments together instead of posting them individually.

    Workflow:
    1. Create comments using |:ReviewComment| (saved as GitHub pending)
    2. Comments appear on the PR page with "Pending" status
    3. Run |:ReviewSubmit| to submit all pending comments as one review
    4. Select review type: Comment, Approve, or Request Changes
    5. Optionally add a review body message
    6. All pending comments are submitted together

    Note: Replies to existing comments and PR-level comments cannot
    be included in a review (GitHub API limitation).

5. KEYMAPS                                                *reviewit-keymaps*

Comment navigation keymaps (]c / [c by default) are set as buffer-local
mappings during review mode and removed on stop. Other keymaps are not
set by the plugin. Configure them in your lazy.nvim spec or manually.
Suggested mappings:

  `<leader>et`  Toggle review mode
  `<leader>es`  Start review mode
  `<leader>eq`  Stop review mode
  `<leader>ec`  Create comment (normal/visual)  *
  `<leader>eS`  Suggest change (normal/visual) *
  `<leader>ev`  View comments on current line
  `<leader>er`  Reply to comment on current line
  `<leader>ef`  List changed files
  `<leader>eo`  PR overview
  `<leader>ed`  Toggle diff preview
  `<leader>eb`  Open PR in browser
  `<leader>el`  List all review comments
  `<leader>ep`  List all drafts
  `]c`          Next comment
  `[c`          Previous comment
  `gx`          Open GitHub reference or URL under cursor (in floats)

  * Visual mode mappings must use `:ReviewComment<CR>` (not `<cmd>`)
    to preserve the range. Using `<C-u>` will clear the range and
    result in single-line comments.

6. CONFIGURATION                                    *reviewit-configuration*

Default configuration: >lua
  require("reviewit").setup({
    -- File list mode: "telescope" or "quickfix"
    file_list_mode = "telescope",
    -- Diff filler character (nil to keep user's default)
    diff_filler_char = nil,
    -- Additional diffopt values applied during review
    diffopt = { "algorithm:histogram", "linematch:60", "indent-heuristic" },
    signs = {
      comment = "#",
      comment_hl = "DiagnosticInfo",
      pending = "⏳ pending",
      pending_hl = "DiagnosticHint",
      draft = "✎ draft comment",
      draft_hl = "DiagnosticWarn",
    },
    float = {
      border = "single",
      -- Width/height as percentage of screen (1-100)
      width = 50,
      height = 50,
    },
    overview = {
      -- Width/height as percentage of screen (1-100)
      width = 80,
      height = 80,
    },
    -- strftime format for timestamps (system timezone)
    date_format = "%Y/%m/%d %H:%M",
  })
<

Options:

`file_list_mode`    "telescope" (default) or "quickfix"
                  Controls which picker `:ReviewFiles` uses.

`diff_filler_char`  Character for diff filler lines (default: nil).
                  Set to " " to hide the filler indicators.

`diffopt`           Table of diffopt values appended during review.
                  Set to nil to keep your existing diffopt settings.

`signs.comment`     Character shown in virtual text for comment indicators.

`signs.comment_hl`  Highlight group for comment indicators.

`signs.pending`     Text shown in virtual text for pending review comments
                  (default: "⏳ pending").

`signs.pending_hl`  Highlight group for pending indicators
                  (default: "DiagnosticHint").

`signs.draft`       Text shown in virtual text for local draft indicators
                  (default: "✎ draft comment"). Local drafts are fallback
                  storage when GitHub API fails.

`signs.draft_hl`    Highlight group for draft indicators
                  (default: "DiagnosticWarn").

`float.border`      Border style for floating windows.

`float.width`       Floating window width as % of screen (default: 50).

`float.height`      Floating window height as % of screen (default: 50).

`overview.width`    Overview window width as % of screen (default: 80).

`overview.height`   Overview window height as % of screen (default: 80).

`auto_view_comment` When true, automatically opens the comment viewer
                  float after navigating to a comment line via ]c, [c,
                  or ReviewListComments (default: true).

`date_format`       strftime format for timestamps (default: "%Y/%m/%d %H:%M").
                  Timestamps from GitHub (UTC) are converted to the system
                  timezone before formatting.

7. COMPLETION                                          *reviewit-completion*

Comment input windows (`ReviewComment`, `ReviewSuggest`) support GitHub
completion for `@user` mentions and `#issue/PR` references.

blink.cmp: >lua
  sources = {
    default = { "lsp", "path", "buffer", "snippets", "reviewit" },
    providers = {
      reviewit = {
        name = "reviewit",
        module = "reviewit.completion.blink",
        score_offset = 50,
        async = true,
      },
    },
  }
<

nvim-cmp: >lua
  require("cmp").register_source("reviewit",
    require("reviewit.completion.cmp").new())
  -- Add { name = "reviewit" } to your sources
<

Data is cached for 5 minutes per session. The first trigger of `@` or `#`
fetches data from the GitHub API asynchronously.
